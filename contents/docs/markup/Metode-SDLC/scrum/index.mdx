<Card>
  <CardTitle>SPRINT 1: Divisi Pengembangan Produk</CardTitle>
  <CardBody>
    <h5>Sprint Planning</h5>
    <p><strong>Tanggal Mulai Sprint:</strong> 1 Jan 2025</p>
    <p><strong>Tanggal Berakhir Sprint:</strong> 30 Mar 2025</p>
    <p><strong>Goal Sprint:</strong> Mengembangkan fitur pencarian produk, sistem pembayaran, fitur keranjang belanja, dan integrasi sistem pengiriman.</p>
    <p><strong>Sprint Backlog:</strong></p>
    <ul>
      <li>Membuat modul pencarian produk</li>
      <li>Implementasi integrasi dengan sistem pembayaran</li>
      <li>Membuat modul keranjang belanja</li>
      <li>Integrasi sistem pengiriman</li>
    </ul>

    <h5>Daily Scrum</h5>
    <p>Setiap hari, tim Scrum akan melakukan rapat singkat untuk mendiskusikan perkembangan, hambatan, dan rencana pekerjaan ke depan. Rapat ini berlangsung selama 15 menit.</p>

    <h5>Sprint Review</h5>
    <p><strong>Hasil:</strong> Fitur pencarian produk berfungsi, sistem pembayaran terintegrasi, fitur keranjang belanja selesai, dan sistem pengiriman terintegrasi.</p>
    <p><strong>Feedback:</strong> Pengguna memerlukan fitur pencarian yang lebih canggih; integrasi pembayaran memerlukan peningkatan keamanan.</p>

    <h5>Sprint Retrospective</h5>
    <p><strong>Yang Berjalan Baik:</strong> Pengembangan fitur keranjang belanja tepat waktu.</p>
    <p><strong>Yang Perlu Ditingkatkan:</strong> Integrasi sistem pembayaran perlu perbaikan pada keamanan.</p>
    <p><strong>Aksi untuk Sprint Berikutnya:</strong> Implementasi langkah-langkah tambahan untuk keamanan integrasi pembayaran.</p>
  </CardBody>
</Card>

<Card>
  <CardTitle>SPRINT 2: Manajemen Kontak Darurat</CardTitle>
  <CardBody>
    <h5>Sprint Planning</h5>
    <p><strong>Tanggal Mulai Sprint:</strong> 31 Mar 2025</p>
    <p><strong>Tanggal Berakhir Sprint:</strong> 30 Apr 2025</p>
    <p><strong>Goal Sprint:</strong> Menyelesaikan manajemen kontak darurat dan pengelolaan database pengguna.</p>
    <p><strong>Sprint Backlog:</strong></p>
    <ul>
      <li>Implementasi fitur CRUD (Create, Read, Update, Delete) untuk kontak darurat.</li>
      <li>Integrasi database MySQL untuk menyimpan data kontak darurat.</li>
      <li>Menambahkan fitur untuk mengelola kontak darurat di dashboard pengguna.</li>
    </ul>

    <h5>Daily Scrum</h5>
    <p>Diskusi perkembangan harian, penyelesaian masalah, dan penyesuaian jadwal untuk pencapaian fitur manajemen kontak darurat.</p>

    <h5>Sprint Review</h5>
    <p><strong>Hasil:</strong> Fitur CRUD untuk kontak darurat berfungsi dengan baik, dan data disimpan dengan aman di database.</p>
    <p><strong>Feedback:</strong> Pengguna membutuhkan pengaturan lebih lanjut untuk mengelompokkan kontak darurat berdasarkan kategori.</p>

    <h5>Sprint Retrospective</h5>
    <p><strong>Yang Berjalan Baik:</strong> Fitur CRUD telah selesai tepat waktu.</p>
    <p><strong>Yang Perlu Ditingkatkan:</strong> Penyimpanan data dapat lebih aman dengan penggunaan enkripsi.</p>
    <p><strong>Aksi untuk Sprint Berikutnya:</strong> Meningkatkan keamanan penyimpanan data menggunakan enkripsi dan penambahan kategori pada kontak darurat.</p>
  </CardBody>
</Card>

<Card>
  <CardTitle>SPRINT 3: Integrasi Notifikasi WhatsApp</CardTitle>
  <CardBody>
    <h5>Sprint Planning</h5>
    <p><strong>Tanggal Mulai Sprint:</strong> 1 Mei 2025</p>
    <p><strong>Tanggal Berakhir Sprint:</strong> 30 Mei 2025</p>
    <p><strong>Goal Sprint:</strong> Integrasi API WhatsApp untuk mengirimkan notifikasi darurat kepada kontak terpilih.</p>
    <p><strong>Sprint Backlog:</strong></p>
    <ul>
      <li>Integrasi API WhatsApp untuk mengirimkan pesan darurat.</li>
      <li>Format pesan otomatis dengan lokasi dan pesan darurat.</li>
      <li>Penambahan tombol untuk mengirim pesan ke setiap kontak.</li>
    </ul>

    <h5>Daily Scrum</h5>
    <p>Tim berdiskusi tentang masalah teknis yang terjadi dalam integrasi API WhatsApp, dan solusi yang akan diterapkan pada fitur notifikasi.</p>

    <h5>Sprint Review</h5>
    <p><strong>Hasil:</strong> Notifikasi darurat dapat dikirim dengan sukses ke nomor WhatsApp yang dipilih.</p>
    <p><strong>Feedback:</strong> Pengguna ingin bisa mengirimkan pesan ke lebih dari satu kontak sekaligus.</p>

    <h5>Sprint Retrospective</h5>
    <p><strong>Yang Berjalan Baik:</strong> Integrasi API WhatsApp berjalan lancar tanpa masalah besar.</p>
    <p><strong>Yang Perlu Ditingkatkan:</strong> Pengguna ingin fitur batch untuk mengirim notifikasi ke banyak kontak sekaligus.</p>
    <p><strong>Aksi untuk Sprint Berikutnya:</strong> Menambahkan fitur untuk mengirim pesan ke banyak kontak sekaligus dan peningkatan UI untuk tombol pengiriman.</p>
  </CardBody>
</Card>
