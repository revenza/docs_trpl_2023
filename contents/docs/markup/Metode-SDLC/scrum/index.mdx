# Metodologi Scrum untuk Aplikasi Notifikasi Darurat dan Blog

## Sprint 0: Inisialisasi Proyek

### Tugas:
1. Identifikasi kebutuhan dan tujuan proyek.
2. Tentukan peran:
   - Product Owner (Pengguna atau Pemangku Kepentingan)
   - Scrum Master
   - Tim Pengembang.
3. Siapkan alat pengembangan (misalnya, Visual Studio Code, Git, server PHP).
4. Tetapkan Product Backlog.
5. Tentukan timeline untuk sprint.

---

## Sprint 1: Registrasi & Autentikasi Pengguna

### Tujuan:
Memungkinkan pengguna untuk mendaftar dan login dengan aman.

### Tugas:
1. **Buat Database:**
   - Tabel `users` dengan kolom: `id`, `username`, `email`, `password`, `created_at`.
2. **Kembangkan Halaman Registrasi:**
   - Pengguna mengisi data mereka dan disimpan di database.
3. **Kembangkan Sistem Login:**
   - Validasi kredensial pengguna dengan database.
4. **Implementasikan Manajemen Sesi:**
   - Kelola sesi pengguna untuk akses dashboard.
5. **Pengujian:**
   - Validasi fungsionalitas registrasi dan login.

### Output yang Diharapkan:
Sistem registrasi dan login yang berfungsi.

---

## Sprint 2: Manajemen Kontak (Operasi CRUD)

### Tujuan:
Memungkinkan pengguna untuk mengelola detail kontak darurat.

### Tugas:
1. **Buat Database:**
   - Tabel `contacts` dengan kolom: `id`, `user_id`, `name`, `phone`, `created_at`, `updated_at`.
2. **Kembangkan Dashboard:**
   - Tampilkan kontak yang ada.
   - Tambahkan tombol untuk membuat kontak baru.
3. **Buat Kontak:**
   - Formulir untuk menambahkan `name` dan `phone`.
   - Simpan data di tabel `contacts`.
4. **Edit Kontak:**
   - Aktifkan pembaruan `name` dan `phone`.
5. **Hapus Kontak:**
   - Hapus kontak dari database.
6. **Pengujian:**
   - Validasi semua operasi CRUD.

### Output yang Diharapkan:
Pengguna dapat mengelola detail kontak darurat di dashboard mereka.

---

## Sprint 3: Integrasi Notifikasi WhatsApp

### Tujuan:
Memungkinkan pengguna untuk mengirim notifikasi darurat ke kontak mereka melalui WhatsApp.

### Tugas:
1. **Integrasi API WhatsApp:**
   - Integrasikan API untuk mengirim pesan.
   - Format pesan dengan lokasi dan detail darurat.
2. **Kembangkan Fitur Kirim Notifikasi:**
   - Tambahkan tombol di samping setiap kontak.
   - Jalankan panggilan API saat tombol diklik.
3. **Pengujian:**
   - Pastikan pesan terkirim dengan benar ke kontak yang dipilih.

### Output yang Diharapkan:
Pengguna dapat mengirim notifikasi darurat ke kontak mereka melalui WhatsApp.

---

## Sprint 4: Implementasi Fitur Blog

### Tujuan:
Memungkinkan pengguna untuk membuat, melihat, dan berinteraksi dengan postingan blog.

### Tugas:
1. **Buat Database:**
   - Tabel `blogs` dengan kolom: `id`, `user_id`, `title`, `image`, `content`, `author`, `created_at`.
2. **Kembangkan Halaman Blog:**
   - Tampilkan semua blog dengan judul dan gambar.
   - Tambahkan tombol untuk membuat blog baru.
3. **Buat Postingan Blog:**
   - Formulir untuk `title`, `image`, `content`, dan `author`.
   - Simpan data di tabel `blogs`.
4. **Lihat Detail Blog:**
   - Tampilkan konten blog lengkap di halaman terpisah.
5. **Pengujian:**
   - Validasi pembuatan dan tampilan blog.

### Output yang Diharapkan:
Sistem blog yang berfungsi penuh.

---

## Sprint 5: Peningkatan UI/UX

### Tujuan:
Meningkatkan pengalaman pengguna dan membuat aplikasi lebih menarik secara visual.

### Tugas:
1. **Terapkan Styling:**
   - Gunakan Bootstrap untuk desain responsif.
2. **Tambahkan Animasi:**
   - Tingkatkan interaktivitas untuk tombol dan formulir.
3. **Uji Kompatibilitas Antar-Browser:**
   - Pastikan tampilan konsisten di berbagai browser.

### Output yang Diharapkan:
Aplikasi yang menarik secara visual dan ramah pengguna.

---

## Sprint 6: Pengujian Akhir dan Deployment

### Tujuan:
Memastikan aplikasi bebas bug dan siap untuk digunakan.

### Tugas:
1. **Pengujian Fungsional:**
   - Verifikasi semua fitur berfungsi sebagaimana mestinya.
2. **Pengujian Keamanan:**
   - Uji kerentanan, misalnya, injeksi SQL.
3. **Pengujian Performa:**
   - Pastikan aplikasi berkinerja baik di bawah beban.
4. **Deployment:**
   - Deploy aplikasi ke server live.
5. **Dokumentasi:**
   - Siapkan panduan pengguna dan pengembang.

### Output yang Diharapkan:
Aplikasi live dan fungsional siap digunakan.
